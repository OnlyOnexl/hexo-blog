测试实验

首先，我们来模拟下”分离头指针”状态的情况。

本来打算用checkout命令来切换分支时，结果后面直接跟上了commit id，结果一下子报了好多log信息，这可怎么怎么办？

“分离头指针”通过官方解释可以看出，其实一把双刃剑。

优点：在”分离头指针”状态下所做的任何commit信息，只要通过git checkout切换到其他分支下，那么之前所做的任何信息均不会被保存在git里面，而是会当做垃圾清理掉，不用我们干预，很方便的，可用于临时测试。

缺点：如果我们处于”分离头指针”状态,当时做了一些测试并进行了commit，但此时需要切换到其他分支做一些修复bug操作，但切换分支后，原来在”分离头指针”状态下所做的任何信息均会被丢弃，这样自己再想利用之前的数据就再也找不回来了，除非在切换后其他分支后，通过git弹出的log信息马上新创建一个分支，这样，处于”分离头指针”状态下的数据又可以找回来了。

总结：

git checkout commitId：会出现分离头指针的情况，这种情况下比较危险，因为这个时候你提交的代码没有和分支对应起来，当切换到其他分支的时候(比如master分支)，容易丢失代码；

但是分离头指针也有它的应用场景，就是在自己做尝试或者测试的时候可以分离头指针，当尝试完毕没有用的时候可以随时丢弃，但是如果觉得尝试有用，那么可以新建一个分支，使用 git branch <新分支的名称> commitId
